Тестовый проект предназначен для демонстрации полного цикла разработки и эксплуатации простого REST API с возможностью автоматизированного развертывания на Ubuntu/Debian-серверах.

REST API (Python/Flask): реализует базовые CRUD-операции для управления пользователями.
Автотесты (Pytest): обеспечивают проверку корректности работы API.
Деплой (Docker Compose): описывает конфигурацию для локального запуска приложения в контейнере.
Автоматизация (Ansible): плейбуки и роли для развертывания приложения на удаленных серверах.
Диагностика (Bash): скрипт для сбора системной информации и проверки доступности сервисов.

В репозитории находятся семь основных файлов, организованных в логические блоки:

-Приложение:

--api.py — основной файл веб-приложения на Python (Flask).

-Тестирование:

--tests.py — набор автотестов для проверки API с помощью Pytest.

-Локальный деплой:

--docker-compose.yml — конфигурация для запуска приложения в Docker.

-Диагностика:

--diagnostics.sh — исполняемый скрипт на Bash для сбора информации о системе и проверки сервисов.

-Автоматизация развертывания (Ansible):

--deploy.yml — главный Ansible-плейбук, связывающий роли и задачи.

--roles/docker/tasks/main.yml — задачи по установке Docker и Docker Compose на сервер.

--roles/app/tasks/main.yml — задачи по развертыванию приложения (создание директории, копирование кода, запуск).



---REST API (Python / Flask)
Назначение: Простой сервис для управления списком пользователей.

Текущие эндпоинты:

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/` | Приветственное сообщение |
| GET | `/health` | Проверка работоспособности |
| GET | `/api/users` | Получить всех пользователей |
| GET | `/api/users/<id>` | Получить пользователя по ID |
| POST | `/api/users` | Создать пользователя |
| DELETE | `/api/users/<id>` | Удалить пользователя |


---Автотесты (Pytest)
Тесты покрывают все основные эндпоинты API, включая проверку валидации данных и обработку ошибок.

Запуск тестов:
Убедитесь, что сервер запущен, и выполните команду в корне проекта:

в терминале - bash выполнить команду : pytest tests.py

Деплой (Docker Compose)
Конфигурация позволяет запустить приложение в изолированном контейнере. В файле настроен healthcheck, который следит за состоянием сервиса через эндпоинт /health.



---Запуск с помощью Docker Compose:

Установите Docker и Docker Compose.

Запустите команду в корне проекта: docker-compose up --build -d

Приложение будет доступно по адресу: http://localhost:5000.

Чтобы остановить контейнеры, выполните: docker-compose down

Для запуска тестов внутри контейнера: docker-compose exec app pytest tests.py



---Автоматизация развертывания (Ansible)

Проект включает готовые роли и плейбуки для автоматизации развертывания на удаленных серверах под управлением Ubuntu/Debian.

Структура ролей:

Роль docker: Устанавливает зависимости, добавляет репозиторий Docker и устанавливает docker-ce и docker-compose. Также добавляет текущего пользователя в группу docker.

Роль app: Создает директорию приложения (/opt/simple-app), копирует туда файлы проекта и запускает его через docker-compose.

Главный плейбук (deploy.yml):

Связывает роли и определяет хосты (webservers), на которых будет происходить развертывание.

Пример использования:

Для запуска плейбука используйте команду: ansible-playbook deploy.yml -i inventory.ini --ask-become-pass

Где inventory.ini — ваш файл инвентаризации с описанием серверов.



  ---Диагностический скрипт (Bash)

Скрипт предназначен для сбора информации о системе и проверки доступности сервисов. Он полезен при отладке и мониторинге.

Использование:

Сделайте скрипт исполняемым: chmod +x diagnostics.sh

Запустите без аргументов для сбора системной информации: ./diagnostics.sh

Запустите с URL-адресами для проверки их доступности: ./diagnostics.sh http://localhost:5000/health http://example.com/health

Для вывода справки используйте флаг: ./diagnostics.sh --help



---Локальный запуск через Docker Compose

Это самый быстрый способ запустить проект на вашей локальной машине.

В корне проекта выполните: docker-compose up --build -d

Проверьте логи контейнера: docker logs -f simple-app # где simple-app — имя контейнера из docker-compose.yml

Для остановки выполните: docker-compose down

Для диагностики запустите скрипт: ./diagnostics.sh http://localhost:5000/health

Для запуска тестов: pytest tests.py # или docker-compose exec app pytest tests.py

Для доступа к приложению откройте в браузере: http://localhost:5000.



---Развертывание на сервере с помощью Ansible

Этот способ используется для развертывания на реальных серверах.

Настройте файл инвентаризации (inventory.ini) с IP-адресами ваших серверов.

Запустите плейбук: ansible-playbook deploy.yml -i inventory.ini --ask-become-pass

После успешного выполнения приложение будет доступно по IP-адресу сервера на порту 5000.
